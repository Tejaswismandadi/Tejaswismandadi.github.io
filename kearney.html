Case Study · Michigan Student Lab Engagement

800+ suppliers, 7 countries, one regulation nobody had mapped yet.

The EU's Carbon Border Adjustment Mechanism (CBAM) requires supplier-level emissions and origin data most manufacturers have never had to track. Nexteer had no documented process for it — this is how we built one.

RoleStudent Consultant
DurationAugust – December 2025
ClientNexteer Automotive
Scope800+ suppliers · 7 countries
01 — Situation

A compliance obligation with no owner and no process.

CBAM requires importers to report embedded emissions on covered goods, with tax exposure calculated per shipment. For Nexteer, that meant tracing CO2 data, CN commodity codes, and weight figures across 800+ suppliers spanning 7 countries — data that had never been centrally collected, let alone validated, and touched five internal functions that had never had to coordinate on it before.

02 — Approach

Map the process before automating it.

Led cross-departmental stakeholder discovery across Engineering, Finance, Purchasing, PC&L, and PwC to document a compliance process that existed nowhere on paper — isolating the specific root causes behind recurring weight and data discrepancies between supplier declarations and Nexteer's own records.

Where supplier data simply didn't exist — common for smaller sub-tier suppliers with no emissions reporting infrastructure — the framework used default emissions values rather than leaving the field blank, so a data gap didn't silently break the compliance calculation downstream.

04 — What Was Built

Mapped the process first — then built validation into it.

Before designing anything, I mapped the existing manual process end to end: Engineering sets part weight and material, Finance assigns the CN code, Purchasing shares it with the supplier, the supplier ships with weight and CN code attached, a customs broker enters that data, and Finance hands it to PwC to calculate and submit. Nowhere in that chain was there a single validation checkpoint — errors surfaced only after submission to the EU, when fixing them was slowest and most expensive.

Mine — current-state process mapNo validation across 7 handoffs; errors caught only after EU submission

The redesign built three validation checkpoints directly into the same process, rather than adding a separate audit step at the end:

  • Pre-shipping weight check — Purchasing's estimated weight is matched against the supplier's actual weight before the shipment ever moves; mismatches route to an error-and-reconcile loop instead of shipping through.
  • Post-shipping weight check — the customs broker's entered weight is checked against the final shipped weight a second time before CBAM applicability is even evaluated.
  • Supplier emissions data check — if a supplier has provided real emissions data, the system uses it; if not, it automatically falls back to default emissions values rather than stalling the calculation, using default emissions values to close data gaps where supplier data was unavailable.
Mine — future-state process map3 validation checkpoints built into the flow, not bolted on after

Built a Power BI dashboard connected directly to the Excel CBAM tool, turning static compliance data into something a compliance or sourcing team could actually interrogate — filterable by country and reporting quarter, with side-by-side supplier comparison on total emissions, shipped weight, and carbon border adjustment cost.

Built separate scenario models linking CBAM cost exposure, regulatory risk, and supplier geography — letting the team stress-test carbon tax liability under different sourcing scenarios rather than just report a single static number.

05 — Outcome

~80% reduction in manual reporting effort.

The validation framework and mapping tool reduced manual compliance reporting effort by roughly 80% — turning a process that had required extensive manual reconciliation across five functions into something a compliance team could run on a repeatable cadence.

Scope note: this was a bounded student consulting engagement (Aug–Dec 2025), separate from the later Nexteer REI risk engagement. The four-phase methodology above is Kearney's standard consulting framework, applied here — not something I originated. The two Nexteer engagements are kept intentionally distinct: CBAM's regulatory-compliance depth doesn't transfer to REI's supplier-risk scoring, and vice versa.
